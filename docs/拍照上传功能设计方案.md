# 拍照上传功能设计方案

## 1. 概述

本方案实现微信小程序拍照上传功能，图片暂时保存到服务器本地文件系统。

## 2. 功能需求

### 2.1 前端功能
- 调用微信相机拍照或从相册选择图片
- 图片压缩处理
- 上传图片到后端 API
- 显示上传进度和结果

### 2.2 后端功能
- 接收图片上传请求
- 图片格式验证和大小限制
- 保存图片到本地文件系统
- 返回图片访问 URL
- 记录上传日志

## 3. 技术方案

### 3.1 前端实现

#### 3.1.1 页面/组件
- 在 `pages/create/index` 页面添加拍照/选择图片功能
- 或创建独立的图片上传组件

#### 3.1.2 核心 API
使用微信小程序 API：
- `wx.chooseMedia()` - 选择图片/拍照
- `wx.compressImage()` - 图片压缩
- `wx.uploadFile()` - 上传文件

#### 3.1.3 图片规格
- 格式：JPG/PNG
- 大小：最大 5MB
- 压缩质量：80%
- 尺寸：最大宽度 1920px

### 3.2 后端实现

#### 3.2.1 API 端点
```
POST /api/v1/upload/image
Content-Type: multipart/form-data
Authorization: Bearer <token>

请求参数：
- file: 图片文件
- type: 图片类型（avatar/card_source/etc）

响应：
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "/uploads/2024/01/15/xxx.jpg",
    "filename": "xxx.jpg",
    "size": 102400
  }
}
```

#### 3.2.2 文件存储结构
```
server/
└── storage/
    └── uploads/
        └── {year}/
            └── {month}/
                └── {day}/
                    └── {uuid}.{ext}
```

#### 3.2.3 静态文件服务
- 配置 Kratos 提供静态文件访问
- 访问路径：`/uploads/*` 映射到 `storage/uploads/`

#### 3.2.4 数据库设计（可选）
暂时不需要数据库表，后续可扩展：
```go
// Image（图片表）
- id
- user_id
- filename
- original_name
- file_path
- file_size
- mime_type
- type
- created_at
```

## 4. 实现步骤

### 4.1 后端实现

1. **创建上传业务层** (`internal/biz/upload.go`)
   - 定义图片保存逻辑
   - 文件验证（格式、大小）

2. **创建上传数据层** (`internal/data/upload.go`)
   - 实现文件系统存储
   - 生成文件路径

3. **创建上传服务层** (`internal/service/upload.go`)
   - 处理 multipart/form-data 请求
   - 调用业务层保存文件

4. **注册路由** (`cmd/server/main.go`)
   - 添加 POST /api/v1/upload/image 路由
   - 配置静态文件服务

### 4.2 前端实现

1. **添加 API 类型定义** (`typings/types/api.d.ts`)
   - 定义上传请求/响应类型

2. **创建上传工具函数** (`utils/upload.ts`)
   - 封装图片选择和上传逻辑

3. **在页面中集成** (`pages/create/index`)
   - 添加拍照/选择图片按钮
   - 显示上传进度和预览

## 5. 安全考虑

- 文件类型验证（仅允许 jpg/jpeg/png）
- 文件大小限制（最大 5MB）
- 文件名随机化（使用 UUID）
- 用户认证（需要登录）
- 后续可添加病毒扫描
