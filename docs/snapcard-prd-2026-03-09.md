## 一、产品定位

**产品名（暂定）：** SnapCard / 拍记

**一句话定位：** 拍一张照片，AI 帮你把知识变成闪卡，科学安排复习时间。

**目标用户：** 任何有定期复习需求的人，包括备考学生（高考/考研/四六级/职业资格证）、职场学习者（语言、技能）。

**核心价值：**
- 零门槛建卡：拍照即生成，省去手动录入
- 科学复习：FSRS 间隔重复算法，记忆效率最优
- AI 辅助判题：不只是"翻卡自评"，支持主观题智能评分

## 五、AI 集成设计

### 5.1 图片分析 → 生成闪卡

**模型：** Qwen3.5（多模态）

**System Prompt 核心指令：**
```
你是一个专业的学习助手。分析用户上传的图片，提取其中的知识点，
并生成结构化的闪卡用于记忆复习。
严格按照以下 JSON 格式输出，不要输出任何其他内容。
```

**输出 JSON 格式：**
```json
{
  "knowledge_points": ["知识点1", "知识点2"],
  "cards": [
    {
      "front": "问题",
      "back": "答案",
      "tags": ["学科", "主题"]
    }
  ]
}
```

**后端处理流程：**
```
接收图片 → base64 编码 → 调用 Qwen3.5 API（vision）
→ 解析 JSON → 写入数据库（card 表，state=New）
→ 返回前端预览
```

### 5.2 AI 判题

**请求体：**
```json
{
  "question": "牛顿第二定律的公式是什么？",
  "standard_answer": "F = ma，即合力等于质量乘以加速度",
  "user_answer": "力等于质量乘加速度"
}
```

**模型返回：**
```json
{
  "correct": true,
  "rating": 3,
  "feedback": "核心概念正确，补充说明这里的 F 指合外力"
}
```

`rating` 映射到 FSRS 的 Again/Hard/Good/Easy，直接驱动调度。

### 5.3 FSRS 调度

使用 `open-spaced-repetition/fsrs-go` 开源库。

每次复习提交后：
```go
card = fsrs.Repeat(card, rating, reviewedAt)
// 更新 due_date, stability, difficulty, reps, lapses, state
db.Save(card)
```

积压保护：若用户连续多日未复习，单次复习上限为 **50 张**，避免打击积极性。

## 九、关键边界 & 错误处理

| 场景 | 处理方式 |
|------|---------|
| AI 未识别到内容（图片模糊/纯图无文字）| 提示"未识别到有效内容，请重新拍照或上传清晰图片" |
| AI 生成 JSON 格式异常 | 后端重试一次，失败则返回"生成失败，请稍后重试" |
| AI 判题接口超时（>5s）| 降级为自评模式，提示"AI 繁忙，请手动自评" |
| 用户连续多天未复习 | 单次复习上限 50 张，其余顺延 |
| 网络中断时进入复习 | 前端缓存当日卡片数据，结果本地存储，恢复网络后上传 |
| 图片含敏感内容 | Qwen3.5 API 层返回拒绝，统一提示"图片内容无法处理" |

## 十、微信订阅消息

- **时机：** 用户首次点击"开始复习"时申请订阅授权
- **推送时间：** 每日 08:00（后端定时任务，Go `cron`）
- **内容模板：** `你今天有 {N} 张闪卡待复习，趁热打铁！`
- **触发条件：** `due_date <= today` 的卡片数 > 0
